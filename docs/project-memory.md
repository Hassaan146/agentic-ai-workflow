# Project Memory

This file stores persistent project decisions that should be respected in future work on Agentic AI Workflow.

## Always-On Skill

The **Decision Maker** skill is now part of the project pipeline.

Use it whenever the project needs:

- frontend planning
- landing page planning
- visual direction
- website copy
- hero image direction
- design prompts
- developer prompts
- GitHub/Vercel launch guidance

## Rule

Before building a polished frontend, the project should pass through the Decision Maker flow:

1. Six decisions:
   - feeling
   - audience
   - anti-audience
   - hero object
   - job
   - three-second memory
2. Three reference buckets:
   - feeling
   - structure
   - detail
3. Three style logics:
   - color
   - typography
   - spatial
4. Four downstream prompts:
   - copywriter
   - visual / 3D
   - design
   - developer
5. Launch guide:
   - GitHub
   - Vercel

## Current Frontend Recommendation

Use **Mission Control Graph** as the first visual direction unless the user later chooses another option.

Reason:

- It communicates orchestration quickly.
- It looks more advanced than a chatbot.
- It matches the backend reality: LangGraph, agent states, traces, verification, and routing.

## Important Constraint

Keep the current GUI simple until the user gives a detailed frontend prompt. Do not over-design the interface early.
